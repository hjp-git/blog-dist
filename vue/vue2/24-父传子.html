组件传值

组件的值是相互独立的，在模板中嵌套的是子组件，外面的是父组件

# 父传子

  • 父传子：父组件向子组件传值
  • 父组件通过绑定属性的方式给子组件传值
  • 子组件通过 props 属性接收， 一般属性值为数组形式来接受多个属性
    • props 可以是属性名，可以是数组形式，也可以是对象形式
    • props 为 对象形式可以指定数据类型，设置默认值

props 接收数据的时候采用了驼峰式命名，html 中需要使用短横线的形式,字符串形式模板没有这个限制

<style scoped> 局部样式生效，防止样式冲突

# 子传父

  • 子传父：子组件向父组件传值
  • 子组件用 this.$emit() 触发事件，传递参数
    this.$emit("自定义事件","传递的数据")
  • 然后父组件用 v-on 监听子组件的事件（就是子组件自定义的事件名称），自定义形参固定为($event)的方法用来接收子组件传递过来的数据
    <HelloWorld @shuo="shuo"/>

@/components/HelloWorld.vue












 









<template>
  <div class="hello">
    <button @click="btn">btn</button>
  </div>
</template>

<script>
export default {
  name: 'HelloWorld',
  methods:{
    btn(){
      this.$emit('shuo','Welcome to HelloWorld')
    }
  }
}
</script>

# 非父子

  • 非父子： 非父子组件之间传值
  • 非父子组件传递数据需要借助于事件中央总线，通过事件中央总线传递数据
    • 提供事件中央总线 const Bus = new Vue();
    • 传递数据方A,通过一个事件触发 Bus.$emit('自定义方法'，'传递的数据')
    • 接收数据方B,在 mounted() 生命周期中触发监听 bus.$on('A组件自定义的方法',callback)
  • 销毁事件,通过 bus.$off() , 销毁之后无法进行数据传递

@/bus.js

import Vue from 'vue'
const Bus = new Vue()
export default Bus

# .sync 修饰符

  • vue 2.3.0+ 新增
  • 作为一个编译时的语法糖存在。它会被扩展为一个自动更新父组件属性的 v-on 监听器
  • 实现子组件与父组件的双向数据绑定

@/components/father.vue

<template>
  <div>
    <div>{{show}}</div>
    <!-- <son v-bind:show="show"  v-on:update:show="show = $event"></son> -->
    <!-- - 缩写 .sync 修饰符 -->
    <son :show.sync="show"></son> 
    <button @click="changeValue">父组件btn</button>
  </div>
</template>

<script>
import son from "@/components/son";
export default {
  components: {
    son,
  },
  data() {
    return {
      show: true,
    };
  },
  methods: {
    changeValue() {
      this.show = !this.show;
    },
  },
};
</script>

export default {

  },

当我们用一个对象同时设置多个 prop 的时候，也可以将这个 .sync 修饰符和 v-bind配合使用

<text-document v-bind.sync="doc"></text-document>
1

这样会把 doc 对象中的每一个 property 都作为一个独立的 prop 传进去，然后各自添加用于更新的 v-on 监听器。

# $attrs & $listeners

  • vue 2.4.0 新增

  • $attrs

    • 包含了父作用域中不作为 prop 被识别(且获取) 的 attribute 绑定。
    • 当一个组件没有声明任何 prop 时，这里会包含所有父作用域的绑定的属性，并且可以通过 v-bind="$attrs" 传入内部组件，在创建高级别的组件时非常有用。

    style class 不作为 prop 被识别(且获取)

  • $listeners

    • 包含了父作用域中的 (不含 .native 修饰器的) v-on 事件监听器。它可以通过 v-on="$listeners" 传入内部组件，在创建更高层次的组件时非常有用。
    • $listeners 就像一个中间件,来接收并使用子组件的监听方法

子组件默认根元素接收传入的属性,设置特殊属性 inheritAttrs:false 可以禁止掉这个行为
