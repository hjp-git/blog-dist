# setup

  • setup 是组合api 的入口函数

  • 新的 option,组件中所用到的数据方法等都要配置在 setup 里,只在 beforeCreate 之前执行一次， this 是 undefined

  • 参数

    • props ：值为对象，包含：组件外部传递过来，且组件内部声明接收了的属性
    • context ：上下文对象
    • attrs : 值为对象，包含：组件外部传递过来，但没有在 props 配置中声明的属性, 相当于 this.$attrs
    • slots : 收到的插槽内容, 相当于 this.$slots
    • emit : 分发自定义事件的函数, 相当于 this.$emit

  • 返回值

    • 对象，模板中直接使用
    • 渲染函数，自定义渲染内容

  • 尽量不要与Vue2.x配置混用
    • Vue2.x配置（data、methos、computed...）中可以访问到setup中的属性、方法。
    • 但在setup中不能访问到Vue2.x配置（data、methos、computed...）。
    • 如果有重名, setup优先。
  • setup 不能是一个 async 函数，因为返回值不再是 return 的对象, 而是 promise , 模板看不到 return 对象中的属性。（后期也可以返回一个 Promise 实例，但需要 Suspense 和异步组件的配合）
<template>
  <div>{{number}}</div>
</template>

<script lang="ts">
import { defineComponent } from 'vue';
export default defineComponent({
  name: 'App',
  setup() {
    const number = 10;
    return {
      number
    }
  },
});
</script>

# ref

  • 作用: 定义基本类型的响应式数据
  • 语法: const xxx = ref(initValue):
    • 创建一个包含响应式数据的引用(reference)对象
    • script 中操作数据: xxx.value
    • 模板中操作数据: 不需要 .value
  • 一般用来定义一个基本类型的响应式数据
<template>
  <div>
    {{count}}
  </div>
  <button @click="update">更新</button>
</template>

<script lang="ts">
import { defineComponent, ref } from 'vue';
export default defineComponent({
  name: 'App',
  setup() {
    // 定义响应式数据
    const count = ref(0)
    const obj = ref({name:'张麻子'})
    // 更新方法
    function update(){
      console.log(obj.value.name);
      count.value++
    }
    return {
      count,
      update
    }
  },
});
</script>

# reactive

  • 作用: 只能定义对象或数组类型的响应式数据
  • const proxy = reactive(obj) ： 接收一个普通对象然后返回该普通对象的响应式代理器对象
  • 响应式转换是深层的：会影响对象内部所有嵌套的属性
  • 内部基于 ES6 的 Proxy 实现，通过代理对象操作源对象内部数据都是响应式的
<template>
  <div>
    <p>名字：{{user.name}}</p>
    <p>年龄：{{user.age}}</p>
    <p>职业：{{user.job}}</p>
    <p>丈夫：{{user.husband.name}}</p>
    <p>丈夫的年龄：{{user.husband.age}}</p>
    <p>丈夫的职位：{{user.husband.job}}</p>
  </div>
  <button @click="update">更新</button>
</template>

<script lang="ts">
import { defineComponent, reactive } from 'vue';
export default defineComponent({
  name: 'App',
  setup() {
    const obj = {
      name: '黎姿',
      age:18,
      job:'演员',
      husband: {
        name: '马廷强',
        age: 33,
        job:'boss'
      }
    }

    // 定义响应式数据 返回的是proxy的代理对象 被代理的目标对象是 obj 对象
    const user = reactive(obj)

    // 更新方法
    function update(){
      console.log(user.name);
      obj.age++
      user.husband.age++
    }

    return {
      user,
      update
    }
  },
});
</script>

# reactive 对比 ref

  • 从定义数据角度对比：
    • ref 用来定义：基本类型数据。
    • reactive 用来定义：对象（或数组）类型数据。

    ref 也可以用来定义对象（或数组）类型数据, 它内部会自动通过 reactive 转为代理对象。

  • 从原理角度对比：
    • ref 通过 Object.defineProperty() 的 get 与 set 来实现响应式（数据劫持）。
    • reactive 通过使用 Proxy 来实现响应式（数据劫持）, 并通过 Reflect 操作源对象内部的数据。
  • 从使用角度对比：
    • ref 定义的数据：操作数据需要 .value ，读取数据时模板中直接读取不需要.value。
    • reactive 定义的数据：操作数据与读取数据：均不需要 .value

# 比较Vue2与Vue3的响应式

# vue2的响应式

  • 原理
    • 对象: 通过 defineProperty 对对象的已有属性值的读取和修改进行劫持(监视/拦截)
    • 数组: 通过重写数组更新数组一系列更新元素的方法来实现元素修改的劫持
Object.defineProperty(data, 'count', {
    get () {}, 
    set () {}
})

  • 问题
    • 对象直接新添加的属性或删除已有属性, 界面不会自动更新
    • 直接通过下标替换元素或更新 length , 界面不会自动更新 arr[1] = {}

# Vue3的响应式

  • 原理
    • 通过Proxy(代理): 拦截对data任意属性的任意(13种)操作, 包括属性值的读写, 属性的添加, 属性的删除等...
    • 通过 Reflect(反射): 动态对被代理对象的相应属性进行特定的操作
  • 文档: Proxy (opens new window) Reflect (opens new window)

Proxy

let person = {
  name:'张三',
  age:18
}

const p = new Proxy(person,{
  get(target,key){
    console.log(target,key);
    return target[key]
  },
  set(target,key,newVal){
    console.log(target,key,newVal);
    target[key] = newVal
  },
  deleteProperty(target,key){
    return delete target[key]
  }
})

# computed 函数

  • 与 Vue2.x 中 computed 配置功能一致
<template>
  <div>
    <p>姓：<input type="text" v-model="person.firstName"></p>
    <p>名：<input type="text" v-model="person.lastName"></p>
    <p>res:{{fullName1}}</p>
    <p>res:{{fullName2}}</p>
  </div>
</template>

<script lang="ts">
import { defineComponent, reactive,computed} from 'vue';
export default defineComponent({
  name: 'DemoTest',
  setup() {
    let person =  reactive({
      firstName:'张',
      lastName:'三',
    })

    // 计算属性简写
    let fullName1 = computed(() => {
      return person.firstName + '-' + person.lastName
    })

    // 计算属性完整
    let fullName2 = computed({
      get(){
        return person.firstName + '-' + person.lastName
      },
      set(value:string){
        const nameArr = value.split('-')
        person.firstName = nameArr[0]
        person.lastName = nameArr[1]
      }
    })

    return {
      person,
      fullName1,
      fullName2
    }
  },
});
</script>

# watch 函数

  • 与Vue2.x中 watch 配置功能一致
  • 两个小“坑”：
    • 监视 reactive 定义的响应式数据时：oldValue 无法正确获取、强制开启了深度监视（deep配置失效）。

    ref 定义的响应式数据 oldValue 可以正确获取

    • 监视 reactive 定义的响应式数据中某个属性时：deep 配置有效。
<template>
  <div>
    <p>sun:{{sum}}</p>
    <p>msg:{{msg}}</p>
    <button @click="sumAdd">sumAdd</button><br/>
    <p>姓名:{{person.name}}</p>
    <p>年龄:{{person.age}}</p>
    <p>职业:{{person.job}}</p>
    <p>副业:{{person.avocation.info}}</p>
    <p>副业工资:{{person.avocation.salary}}k</p>
    <button @click="personChange">personChange</button>
  </div>
</template>

<script lang="ts">
import { defineComponent,reactive,ref,watch} from 'vue';
export default defineComponent({
  name: 'DemoTest',
  setup() {
    let sum = ref(0)
    let msg = ref('你好')
    let person = reactive({
      name:'陈百详',
      age:18,
      job:'配角演员',
      avocation:{
        info:'美团外卖',
        salary:10
      }
    })

    function sumAdd():void{
      sum.value++;
      msg.value += sum.value
    }

    function personChange():void{
      person.age++
      person.name = '某某某'
      person.avocation.salary++
    }

    // 监视 一个 ref 定义的响应式数据
    watch(sum,(newValue,oldValue)=>{
      console.log('sum变化了',newValue,oldValue)
    },{
      // immediate: true,  // 是否初始化立即执行一次, 默认是false
      // deep: true, // 是否是深度监视, 默认是false
    })

    // 监视多个 ref 定义的响应式数据
    watch([sum,msg],(newValue,oldValue)=>{
      console.log('sum或msg变化了',newValue,oldValue)
    })

    // 监视 reactive 定义的响应式数据
    watch(person,(newValue,oldValue)=>{
      console.log('person变化了',newValue,oldValue)
    },{deep:false}) // 此处的deep配置不再奏效


    // 监视 reactive 定义的响应式数据中的某个属性
    watch(()=>person.age,(newValue,oldValue)=>{
      console.log('person的age变化了',newValue,oldValue)
    },{deep:true})

    // 监视 reactive 定义的响应式数据中的某些属性
    watch([()=>person.job,()=>person.name],(newValue,oldValue)=>{
      console.log('person的job或者name变化了',newValue,oldValue)
    },{deep:true})

    // 特殊情况 深度监听
    watch(()=>person.avocation,(newValue,oldValue)=>{
      console.log('person的avocation变化了',newValue,oldValue)
    },{deep:true}) //此处由于监视的是reactive素定义的对象中的某个属性，所以deep配置有效

    return {
      sum,
      msg,
      person,
      sumAdd,
      personChange
    }
  },
});
</script>

# watchEffect 函数

  • watch ：既要指明监视的属性，也要指明监视的回调。

  • watchEffect ：不用指明监视哪个属性，监视的回调中用到哪个属性，那就监视哪个属性。 (智能监视)

  • watchEffect 有点像 computed

    • 但 computed 注重的计算出来的值（回调函数的返回值），所以必须要写返回值。
    • 而 watchEffect 更注重的是过程（回调函数的函数体），所以不用写返回值。
<template>
  <div>
    <p>sun:{{sum}}</p>
    <button @click="sumAdd">sumAdd</button><br/>
  </div>
</template>

<script lang="ts">
import { defineComponent, ref,watch,watchEffect} from 'vue';
export default defineComponent({
  name: 'DemoTest',
  setup() {
    let sum = ref(0)
    function sumAdd():void{
      sum.value++;
    }

    watch(sum,(newValue,oldValue)=>{
      console.log('sum变化了',newValue,oldValue)
    })

    watchEffect(()=>{
      console.log('watchEffect回调执行');
      const x1 = sum.value
      console.log(x1);
    })

    return {
      sum,
      sumAdd,
    }
  },
});
</script>

# 生命周期

  • vue3.0中可以继续使用Vue2.x中的生命周期钩子，但有有两个被更名：
    • beforeDestroy 改名为 beforeUnmount
    • destroyed 改名为 unmounted
  • Vue3.0也提供了 Composition API 形式的生命周期钩子，与Vue2.x中钩子对应关系如下：
    • beforeCreate===>setup()
    • created=======>setup()
    • beforeMount ===>onBeforeMount
    • mounted=======>onMounted
    • beforeUpdate===>onBeforeUpdate
    • updated =======>onUpdated
    • beforeUnmount ==>onBeforeUnmount
    • unmounted =====>onUnmounted

# 自定义hook函数

  • 什么是 hook ？ —— 本质是一个函数，把 setup 函数中使用的 Composition API 进行了封装。
  • 类似于vue2.x中的 mixin 。
  • 自定义 hook 的优势: 复用代码, 让 setup 中的逻辑更清楚易懂。

./src/hooks/usePoint.ts

import {
  reactive,
  onMounted,
  onBeforeUnmount
} from 'vue'

export default function () {
  // 变量
  const point = reactive({
    x: 0,
    y: 0
  })
  // 方法
  function savePoint (event: any) {
    point.x = event.pageX
    point.y = event.pageY
  }
  // 钩子
  onMounted(() => {
    window.addEventListener('click', savePoint)
  })
  onBeforeUnmount(() => {
    window.removeEventListener('click', savePoint)
  })
  return point
}

# toRef 与 toRefs

  • 作用：创建一个 ref 对象，其 value 值指向另一个对象中的某个属性。
  • 语法：const key = toRef(object,'key') & toRefs(object)
  • 应用: 要将响应式对象中的某个属性单独提供给外部使用时。

toRefs 与 toRef 功能一致，但可以批量创建多个 ref 对象

# toRef

<template>
  <div>
    <p>姓名:{{name}}</p>
    <p>年龄:{{age}}</p>
    <p>职业:{{job}}</p>
    <p>副业:{{info}}</p>
    <p>副业工资:{{salary}}k</p>
    <button @click="age++">年龄</button><br/>
    <button @click="salary++">副业工资</button>
  </div>
</template>

<script lang="ts">
import { defineComponent,reactive,toRef} from 'vue';
export default defineComponent({
  name: 'DemoTest',
  setup() {
    let person = reactive({
      name:'陈百详',
      age:18,
      job:'配角演员',
      avocation:{
        info:'美团外卖',
        salary:10
      }
    })

    let name = toRef(person,'name')
    let age = toRef(person,'age')
    let job = toRef(person,'job')
    let info = toRef(person.avocation,'info')
    let salary = toRef(person.avocation,'salary')

    return {
      name,
      age,
      job,
      info,
      salary,
    }
  },
});
</script>
