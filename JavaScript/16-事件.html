# 事件初识

什么是JS事件：当HTML页面中使用JS的时候，JS处理(应对)的行为

什么是HTML事件：发生在HTML元素上的事情就叫HTML事件

事件处理程序：响应某个事件的函数，比如 btn.onclick = function(){}，元素支持的事件，都可以使用与响应事件处理程序同名的HTML特性来指定

dom对象.事件 = function(){
  // 代码块
};

# 事件分类

# 鼠标事件

  • onclick 单击
  • ondblclick 双击
  • onmouseover/onmouseout 移入/移出
  • onmouseenter/onmouseleave 移入/移出
  • onmousemove 在元素上移动（会一直触发）
  • onmousedown/onmouseup 鼠标按下/抬起
  • oncontextmenu 阻止右键菜单
  • onselectstart 阻止文本选中

# 键盘事件

  • onkeydown 按键按下
  • onkeyup 按键抬起
  • onkeypress 按键按住不松手

# 表单事件

  • onfocus 获取焦点
  • onblur 失去焦点
  • onchange 表单value值改变时，且元素失去焦点
  • oninput 用户输入时触发，该事件类似于onchange事件，不同之处在于oninput事件在元素值发生变化是立即触发，onchange在元素失去焦点时触发，另外一点不同是 onchange 事件可以可以作用于 select 元素
  • onselect 在input文本框中的文本被选中时发生
  • onsubmit 表单提交时发生
  • onreset 表单重置时发生

# UI事件(用户交互有关的事件)

  • onload 加载事件
  • onscroll 滚动条事件
  • onresize 窗口事件

# 滚轮事件

  • onmousewheel 鼠标滚轮事件
function addWheel(obj,fn){
  function wheel(event){
    var event = event||window.event;
    var bDown = true;
    if(event.wheelDelta){
      if(event.wheelDelta < 0){
        bDown = false;
      }else{
        bDown = true;
      }
    }else{
      if(event.detail < 0){
        bDown = true;
      }else{
        bDown = false;
      }
    }
    fn(bDown);
    // return false;
    // oEvent.preventDefault&&oEvent.preventDefault();
  }
  //判断是不是火狐
  if(window.navigator.userAgent.toLowerCase().indexOf('firefox') != -1){ 
    //'DOMMouseScroll'只兼容火狐,必须用事件绑定;   DOM 2级
    obj.addEventListener('DOMMouseScroll',wheel,false);
  }else{
    obj.onmousewheel = wheel;   // DOM 0级
  }
}

  }

# 事件流

事件流：页面触发一个事件时，会按照一定的顺序来响应事件，事件的响应过程称为事件流

事件流分为三大阶段

  1. 捕获阶段：事件会从docuemt开始，一级一级往里传递，如果在传递的过程中DOM节点有绑定的捕获类型的处理函数，就会执行
  2. 目标阶段：目标元素正在触发事件处理函数的阶段
  3. 冒泡阶段：目标元素从里到外开始，一级一级往外传递，如果在传递的过程中DOM节点有绑定的冒泡类型的处理函数，就会执行

# 事件对象Event

Event对象代表事件的状态，比如事件在其中发的元素，键盘按键的状态，鼠标的位置，鼠标的按钮的状态

什么时候会产生Event对象呢？

例如：当用户单击某个元素的时候，这个元素绑定的事件就会触发

该事件的本质上就是一个函数，而该函数的形参接收一个Event对象

事件通常与函数结合，函数不会在事件发生前被执行。

在触发事件的函数里，会接收一个Event对象，通过该对象，我们可以得到需要的参数，比如键盘按键的状态，鼠标的位置，鼠标的按钮的状态，或者阻止浏览器默认行为，或者事件的冒泡等

  • 获取js的事件对象 Event，IE9以下浏览器兼容写法
var event = event || window.event
  • 阻止默认行为，IE9以下浏览器兼容写法
event.preventDefault();		//IE9及以上写法
event.returnValue = false;	//IE9以下写法

//兼容写法
//console.log(event.preventDefault);goole  fn(true)   IE undefined(false)
event.preventDefault ? event.preventDefault() : event.returnValue = false;
  • 阻止事件冒泡，E9以下浏览器兼容写法
event.stopPropagation();	//IE9及以上写法
event.cancelBubble = true;	// IE9以下兼容写法

//兼容写法
//console.log(event.stopPropagation); goole fn(true   IE undefined(false)
event.stopPropagation ? event.stopPropagation() : event.cancelBubble = true;

# 执行机制

遇到同步直接放到执行栈中执行，遇到异步生成对应的回调放在任务队列中挂起，等执行栈中同步执行完了后，再把任务队列中的回调轮询放在执行栈中执行
主线程(执行栈) ==> 微任务 ==> 宏任务

# 宏任务 & 微任务

任务队列中分为 宏任务 和 微任务，微任务 优先 宏任务执行，宏任务等待微任务
宏任务是由宿主（浏览器、Node）发起的，而微任务由 JS 自身发起。

requestAnimationFrame ✅ ❌
微任务 浏览器 Node
process.nextTick ❌ ✅
MutationObserver ✅ ❌
Promise.then Promise.catch Promise.finally ✅ ✅

# HTML事件缺点

1.存在一定的时差，用户可能在页面一出现就触发相应的事件，但是事件处理程序尚不具备执行的条件。
比如，button的click函数定义在button的下方，我们在函数未解析之前，就点击button，这样就会导致报错

2.这样的事件处理程序的作用域链在不同的浏览器中会导致不同结果，不同的js引擎遵循的标识符解析规则略有差异，很可能在访问非限定对象时出错

3.HTML与JavaScript代码的紧密耦合，如果要更换事件或者事件处理程序就需要更改两个地方，非常不方便

# DOM 0级事件缺点

在JavaScript中通过on的形式绑定的事件，它有一个缺点是如果多次给同一个元素绑定同一个事件，那么后面的事件就会覆盖前面的事件

// DOM0级事件解除绑定    把这个绑定的事件赋值为null
h1.onclick = null;

# DOM 2级 事件

DOM2级事件处理程序定义了两个方法，用于添加事件处理程序和删除事件处理程序的操作

addEventListener("事件名称",callbck,Boolean)         // 添加事件
removeEventListener('事件名称',callbck,Boolean)      // 删除事件

// 布尔值，默认值false，布尔值如果为true，表示在捕获阶段指向事件处理程序，如果为false，表示在冒泡阶段调用事件处理程序

1.使用addEventListener()添加事件，必须使用removeEventListener()才能移除

2.事件处理程序的函数必须是命名函数，不然使用removeEventListener()移除不掉

3.对于IE8及IE8以下浏览器来说，他们没有上述的两个方法,但是提供了 attachEvent 和 detachEvent 两个方法

这个两个只需要传递两个参数：第一个参数事件程序名称，第二个参数事件程序处理函数

// 跨浏览器事件处理程序的兼容写法：
// 添加事件
function addEvent(element,type,fn){
    if (element.addEventListener) {  // 非IE8以上的浏览器
        return element.addEventListener(type,fn,false);
    }else if(element.attachEvent){
        //onclick,fn
        return element.attachEvent('on'+ type,fn);
    }else{
        element['on'+type] = fn;
    }
}
// 移除事件
function removeEvent(element,type,fn){
    if (element.removeEventListener) {  // 非IE8以上的浏览器
        return element.removeEventListener(type,fn,false);
    }else if(element.detachEvent){
        //onclick,fn
        return element.detachEvent('on'+ type,fn);
    }else{
        element['on'+type] = null;
    }
}
