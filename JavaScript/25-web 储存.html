web 储存

# 什么是web存储

  • web存储指的是在web通信过程中，由客户端（如浏览器）对少量数据进行的本地存储（注：本文所说的是广义的web存储，而不单指Web Storage）。

  • web存储类型主要包括古老的 cookie ，以及HTML5推出的 sessionStorage localStorage 和 indexedDB 等。

  • web存储分为 cookie 和 webStorage 存储
    而 webStorage 存储分为 sessionStorage（会话存储）和 localStorage（本地存储）。

# cookie

  1. cookie 的描述：当web服务器向浏览器发送web页面时，在连接关闭之后，服务端不会记录用户的信息

  2. cookie 的定义：cookie 是一些数据，存储在你电脑上的文件中

  3. cookie 的作用：用于解决“如何记录客户端的用户信息”

  • 当用户访问web页面时，它的名字可以记录在cookie中
  • 在用户下一次访问该页面时，可以在cookie中读取用户访问记录

# cookie的特点

  • cookie保存在客户端的一个小文档，每次通信的时候传递给服务端，服务端根据通信进行一系列操作（比如判断用户是否登录状态）

  • cookie一般是服务端要求客户端记录的一部分数据（类似于门禁卡），cookie是有过期时间的（默认的过期时间是在页面关闭之前），过期之后自动删除，依赖于本地电脑删除一个文件

  • cookie只能在相同域名之间的不同页面进行数据交互，不能在不同的域名进行交互

  • cookie的数据是在每一次进行数据交互的时候（或者数据请求的时候）自动携带自动传输

# cookie的局限性

  • 每一次请求都会自动带上cookie的所有数据，会影响网站的性能

  • cookie保存的数据不能太大，一般大小限制4kb（大概2000汉字）

  • cookie是字符串的形式存储在本地，操作非常不方便（特别是用js操作cookie）

# cookie的属性

Name 键，比如 username = zhangsan
Value 值，比如 username = zhangsan
expires 绝对的过期时间,比如 new Date()
domain 限定域名,比如www.baidu.com
path 限定路径 /index.html
max-Age 相对失效时间，单位为秒，取代expires，冲突以max-Age为准
secure 协议，不需要指定，当HTTPS通信时自动打开
HttpOnly 仅用于服务器发送，JS无法读取

# 设置 cookie 封装函数

cookie 的值本质上是一个字符串，不管有多少个 cookie ，都保存在一个字符串里，每一个值都是以键值形式存在的，使用分号隔开

"key1=value1;key2=value2;key3=value3"
"name=zhangsan;age=20;sex=男"
1
2
/*
c_name : 代表名称（键名）
value : 代表值 （键值） 
expiredays : 代表失效的天数（可选）
*/
function setCookie(c_name,value,expiredays){
// console.log(expiredays);
var time = new Date();
var getDay = time.getDate() + expiredays;
time.setDate(getDay);
document.cookie = c_name +'='+ value + (expiredays=='undefined'?'':';expires=' + time.toGMTString());
}

setCookie('LOL','yasuo',7);
setCookie('LOL','yasuo');

# 删除 cookie 封装函数

原理：时间过期自动删除

/*
c_name : 代表名称（键名）
*/
function deleteCookie(c_name){
var time = new Date();
var getDay = time.getDate() - 1;
time.setDate(getDay);
document.cookie = c_name +'= 1;expires='+ time.toGMTString();
}

function getCookie(c_name) {
    var cookieArr = document.cookie.split("; ");//使用"; "分割Cookie
    //循环的得到Cookie名称与值
    for (i = 0; i < cookieArr.length; i++) {
        var  temp = cookieArr[i].split("=");//用"="分割Cookie的名称与值
        if(temp[0]==c_name){
        	return temp[1];
        }
    }
}

# localStorage

  1. localStorage 的定义：用于长久保存整个网站的数据，保存的数据没有过期时间，除非手动清除。大小限制（5Mb），它和cookie不同之处在于不会主动传输。

  2. localStorage 的兼容：IE9以下浏览器不支持web存储，可以使用cookie

  3. localStorage 的优势：数据存储量大，保存数据的时间长，方便操作

# localStorage 创建

localStorage.setItem(key,value);
localStorage.setItem('name','yasuo');

# sessionStorage

  1. sessionStorage 的定义：用于临时保存整个网站的数据，保存的数据过期时间为关闭网站之前。大小限制（5Mb），它和cookie不同之处在于不会主动传输。

  2. sessionStorage 的兼容：IE9以下浏览器不支持web存储，可以使用 cookie

  3. sessionStorage 的优势：数据存储量大，方便操作
