正则表达式

  • 正则表达式（Regular Expression）定义：是一种文本模式，包括普通字符（字母a-z，数字0-9）,特殊字符（$ . !）
  • 正则表达式作用：使用单个字符串来描述、匹配 一系列匹配某个句法规则的字符串
  • 字符串调用RegExp对象的方法或String对象的方法来匹配字符串中指定的字符
  • 正则表达式很繁琐，但是它也很强大，主要的应用场景：评论，聊天，写作

通过正则表达式，可以:

  • 测试字符串内的模式：可以测试输入字符串，查看字符串内是否出现电话卡或信用卡号码模式，这种称为数据验证
  • 替换文本：可以使用正则来识别文档中的特定文本，完全删除该文本或者用其它的文本将它替换
  • 基于模式匹配从字符串中提取子字符串：可以查找文档内或输入域内特定的文本

# 定义正则表达式

修饰符 描述
i 执行对大小写不敏感的匹配
g 是否执行全局匹配（查找所有匹配而非在找到第一个匹配后停止）
m 执行多行匹配

# 字面量定义

var reg = /patten/modifiers;
// patten(模式): 描述了表达式的规则（模式）  
// modifiers(修饰符) 修饰符用"/"隔开，也可以合并在一起

var reg = /a/g; // 全局匹配 'a'

# 构造函数定义

var reg = new RegExp(patten,modifiers);
//patten(模式): 描述了表达式的规则（模式）
//modifiers(修饰符)

var reg = new RegExp('a','g'); // 全局匹配 'a'

# 普通字符

普通字符包括没有显式指定为元字符的所有可打印和不可打印字符。这包括所有大写和小写字母、所有数字、所有标点符号和一些其他符号。

# 特殊字符

所谓特殊字符，就是一些有特殊含义的字符
正则表达式按大类分普通字符和特殊字符，按功能分括号，元字符，限定符（量词）

特别字符 描述
$ 匹配输入字符串的结尾位置。如果设置了 RegExp 对象的 Multiline 属性，则 $ 也匹配 '\n' 或 '\r'。要匹配 $ 字符本身，请使用 $。
( ) 标记一个子表达式的开始和结束位置。子表达式可以获取供以后使用。要匹配这些字符，请使用 ( 和 )。
* 匹配前面的子表达式零次或多次。要匹配 * 字符，请使用 *。
+ 匹配前面的子表达式一次或多次。要匹配 + 字符，请使用 +。
. 匹配除换行符 \n 之外的任何单字符。要匹配 . ，请使用 . 。
[ ] 匹配某个范围内的字符 [0-9] [a-z]。要匹配 [，请使用 [。
? 匹配前面的子表达式零次或一次，或指明一个非贪婪限定符。要匹配 ? 字符，请使用 ?。
\ 将下一个字符标记为或特殊字符、或原义字符、或向后引用、或八进制转义符。例如， 'n' 匹配字符 'n'。'\n' 匹配换行符。序列 '' 匹配 ""，而 '(' 则匹配 "("。
^ 匹配输入字符串的开始位置，除非在方括号表达式中使用，当该符号在方括号表达式中使用时，表示不接受该方括号表达式中的字符集合。要匹配 ^ 字符本身，请使用 ^。
{ 限定每次匹配的字符串长度。要匹配 {，请使用 {。
| 指明两项之间的一个选择。要匹配 |，请使用 |。

# 中括号详情

表达式 描述
[abc] 查找方括号之间的任何字符
[^abc] 查找任何不在方括号之间的字符
[0-9] 查找任何0 至9 的数字
[a-z] 查找任何从小写a 到小写z 的字符
[A-Z] 查找任何从大写A 到大写Z 的字符
[A-z] 查找任何从大写A 到小写z 的字符 - 所有的大小写英文字母
[adgk] 查找给定集合内的任何字符
[^adgk] 查找给定集合外的任何字符
(red|blue|green) 查找任何指定的选项

# 非打印字符

非打印字符也可以是正则表达式的组成部分。下表列出了表示非打印字符的转义序列

字符 描述
\cx 匹配由x指明的控制字符。例如， \cM 匹配一个 Control-M 或回车符。x 的值必须为 A-Z 或 a-z 之一。否则，将 c 视为一个原义的 'c' 字符。
\f 匹配一个换页符。等价于 \x0c 和 \cL。
\n 匹配一个换行符。等价于 \x0a 和 \cJ。
\r 匹配一个回车符。等价于 \x0d 和 \cM。
\s 匹配任何空白字符，包括空格、制表符、换页符等等。等价于 [ \f\n\r\t\v]。注意 Unicode 正则表达式会匹配全角空格符。
\S 匹配任何非空白字符。等价于。
\t 匹配一个制表符。等价于 \x09 和 \cI。
\v 匹配一个垂直制表符。等价于 \x0b 和 \cK。

# 元字符

元字符（Metacharacter）是拥有特殊含义的字符

元字符 描述
. 查找单个字符，除了换行（\n）和行结束符
\w 查找单词字符 -- 相当于大小写字母加0-9数字加下划线_加$符号 [A-z][0-9]_$
\W 查找非单词字符
\d 查找数字 -- 相当于 [0-9]
\D 查找非数字字符
\s 查找空白字符
\S 查找非空白字符
\b 匹配单词边界
\B 匹配非单词边界
\0 查找 NULL 字符
\n 查找换行符
\f 查找换页符
\r 查找回车符
\t 查找制表符
\v 查找垂直制表符
\xxx 查找以八进制xxx 规定的字符
\xdd 查找以十六进制数 dd 规定的字符
\uxxxx 查找以十六进制数 xxx规定的Unicode字符
// 匹配中文

[\u4E00-\u9FA5\\s]+   // 多个汉字，包括空格

[\u4E00-\u9FA5]+      // 多个汉字，不包括空格

[\u4E00-\u9FA5]       // 一个汉字

# 限定符（量词）

量词 描述
\ 将下一个字符标记为一个特殊字符、或一个原义字符、或一个向后引用、或一个八进制转义符。例如，“n”匹配字符“n”。“\n”匹配一个换行符。串行“\\”匹配“\”而“\(”则匹配“(”。
n+ 匹配任何包含至少一个 n 的字符串。例如，/a+/ 匹配 "candy" 中的 "a"，"caaaaaaandy" 中所有的 "a"。
n* 匹配任何包含零个或多个 n 的字符串。例如，/bo*/ 匹配 "A ghost booooed" 中的 "boooo"，"A bird warbled" 中的 "b"，但是不匹配 "A goat grunted"。
n? 匹配任何包含零个或一个 n 的字符串。例如，/e?le?/ 匹配 "angel" 中的 "el"，"angle" 中的 "le"。
n{X} 匹配包含 X 个 n 的序列的字符串。例如，/a{2}/ 不匹配 "candy," 中的 "a"，但是匹配 "caandy," 中的两个 "a"，且匹配 "caaandy." 中的前两个 "a"。
n{X,} X 是一个正整数。前面的模式 n 连续出现至少 X 次时匹配。例如，/a{2,}/ 不匹配 "candy" 中的 "a"，但是匹配 "caandy" 和 "caaaaaaandy." 中所有的 "a"。
n{X,Y} X 和 Y 为正整数。前面的模式 n 连续出现至少 X 次，至多 Y 次时匹配。例如，/a{1,3}/ 不匹配 "cndy"，匹配 "candy," 中的 "a"，"caandy," 中的两个 "a"，匹配 "caaaaaaandy" 中的前面三个 "a"。注意，当匹配 "caaaaaaandy" 时，即使原始字符串拥有更多的 "a"，匹配项也是 "aaa"。
n$ 匹配任何结尾为 n 的字符串。
^n 匹配任何开头为 n 的字符串。
?=n 匹配任何其后紧接指定字符串 n 的字符串。

# 使用正则表达式

# RegExp对象的方法

RegExp.test('str') // 检索字符串中指定的值，返回true或false

var str = "sdfdsfdsfsdfdsgg";
var reg1 = new RegExp("d", "g")
var reg2 = new RegExp("a", "g")

console.log(reg1.test(str)); //true
console.log(reg2.test(str)); //false

RegExp.exec('str')
// 检索字符串中第一个指定的值（正则），返回的是一个数组，检索不到返回null
// 数组中包含：第一个匹配到的值,对应的下标 ,字符串 

var str = "sdfdsfdsfsdfdsgg";
var reg1 = new RegExp("d", "g")
var reg2 = new RegExp("a", "g");

console.log(reg1.exec(str));
//["d", index: 1, input: "sdfdsfdsfsdfdsgg", groups: undefined]
console.log(reg2.exec(str)); //null

RegExp.compile(regexp,modifier) // 在1.5版本中已废弃，编译正则表达式

var reg2 = /o/g;
var str1 = 'hello world!';
var str2 = 'l';

var patt = reg2.compile(str2);   //   /l/
console.log(patt.test(str1));

# String对象的方法

String.search(RegExp) // 返回与正则表达式第一个相匹配的值对应的下标，如果没有找到，返回 -1

var str = "hello word"
var reg2 = new RegExp("o", "g");

console.log(str.search(reg2)); //4

String.replace(RegExp,str) // 替换与正则表达式匹配的子字符串，返回新字符串，不改变原字符串

var str = "hello word"
var reg2 = new RegExp("o", "g");

console.log(str.replace(reg2, "b")); // hellb wbrd

String.match(RegExp) // 找到一个或多个正则表达式的匹配的值，返回的是数组，如有没有，返回null

var str = "hello worood"
var reg1 = new RegExp("o", "g");
var reg2 = new RegExp("a", "g");

console.log(str.match(reg1));	//["o", "o", "o", "o"]
console.log(str.match(reg2));	//null

String.split(RegExp) // 把字符串分割为字符串数组，返回数组，不改变原字符串

var str = "hello word"
var reg2 = new RegExp("o", "g");

console.log(str.split(reg2));	//["hell", " w", "rd"]

# 贪婪、懒惰、占有

  • 贪婪模式
    量词会首先匹配整个字符串，尝试匹配时，会尽可能选定更多的内容，量词首次匹配整个字符串，如果失败则回退一个字符后再尝试，这个过程叫回溯，它每次回退一个字符，直到找到匹配内容或者没有字符尝试为止 (+,*)

  • 懒惰模式
    它从目标的起始位置开始尝试寻找匹配，每次检查字符串的一个字符，最后他会尝试匹配整个字符串，要是一个量词成为懒惰的，必须在普通量词的后面加一个问号？ (+? *?)

  • 占有量词
    它会覆盖整个目标然后尝试寻找匹配内容，但他只尝试一次，不会回溯。占有量词是在普通量词之后添加一个加号 +

# 优先级

  • 正则表达式从左到右进行计算，并遵循优先级顺序，这与算术表达式非常类似
  • 相同优先级的会从左到右进行运算，不同的优先级运算先高后低

依次从最高到最低 - 正则表达式运算符的优先级顺序

  1. \ => 转义符
  2. （），（？）,（？=），[] => 小括号和中括号
      • ? {n} {n,} {n,m} => 量词限定符
  4. ^ $ => 量词限定符
  5. | => ‘或’操作
