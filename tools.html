# 自动生成文件目录结构

mddir 生成目录结构，默认会忽略依赖包

  • mac 安装需求权限 sudo npm install mddir -g

  • windows 安装 npm install mddir -g

安装完毕，直接进入项目根目录，执行命令 mddir 即可在根目录下生成一个 directoryList.md 的文件。

# 删除 node_modules

删除 项目下的 node_modules ，总提示需要管理员权限，右键获取管理员权限后，还是删不了，下面方法完美解决
npm install rimraf -g
rimraf node_modules

# npm 切换镜像源

nrm 是一个 npm 的源管理器，可以快速的在 nrm 的各个源之间切换，方便快捷。

  • 全局安装 npm i nrm -g or yarn global add nrm
  • 查看所有可用镜像源 nrm ls
  • 切换源 nrm use <name>
  • 测试源的速度 nrm test
  • 添加源 nrm add baidu https://registry.npm.baidu.com
  • 删除源 nrm del baidu

# yarn 切换镜像源

yrm 是一个 yarn 的源管理器，可以快速的在 yarn 的各个源之间切换，方便快捷。

  • 全局安装 nnpm install -g yrm or yarn global add yrm
  • 查看所有可用镜像源 yrm ls
  • 切换源 yrm use <name>
  • 测试源的速度 yrm test
  • 添加源 yrm add baidu https://registry.npm.baidu.com
  • 删除源 yrm del baidu

# node 版本切换

NVM 实现多版本共存和无缝切换 node.js
视频教程 (opens new window)
博客地址 (opens new window) 注意是 settings.txt 文件，博客这里有误
版本参考 (opens new window)

版本号逢单 测试版本 逢双都是稳定版本

配置完 nvm

  • 安装指定版本 nvm install xx.xx.xx

  • 设置 node 全局安装的 储存位置

    • npm config set prefix "D:\node\node_global"
    • npm config set cache "D:\node\node_cache"

  • 配置 系统的环境变量(path) D:\node\node_global

  • 查看已安装版本 nvm list

  • 查看可安装版本 nvm list available

  • 激活指定版本（激活需要管理员命令窗口）nvm use xx.xx.xx
